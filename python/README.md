# 2. 파이썬(PC/SBC) 단계: 고성능 연산 및 데이터 분석
  - 파이썬은 연산 능력은 좋지만 하드웨어 직접 제어에는 지연이 발생하므로, 두뇌 역할을 하는 복잡한 알고리즘에 적합합니다.
  - 칼만 필터 (Extended/Unscented Kalman Filter)
    근거: 칼만 필터는 행렬 연산이 많아 아두이노에서는 메모리와 연산 속도 부족으로 구현이 까다롭습니다. 파이썬의 NumPy 라이브러리를 쓰면 훨씬 정밀하고 복잡한 필터링을 쉽게 처리할 수 있습니다.
  - 경로 계획 및 슬램 (Path Planning & SLAM)
    근거: IMU 데이터를 누적하여 지도를 그리거나 다음 이동 경로를 최적화하는 알고리즘(A*, Dijkstra 등)은 많은 메모리를 요구합니다. 이는 아두이노의 SRAM 용량으로는 감당하기 어렵습니다.
  - 머신러닝 기반 주행 알고리즘
    근거: 센서 데이터를 학습시켜 "이 정도 흔들림과 밝기 변화일 때는 어떤 노면 상황이다"라고 판단하는 추론 모델은 파이썬 환경(TensorFlow, PyTorch)에서만 원활하게 구동됩니다.
    알고리즘 추천 :
      1. k-NN(k-최근접 이웃): 장점은 쉬운 알고리즘이다. 단점은 RAM 요구가 많다(데이터셋의 저장이 필요함)
      2. Random Forest(랜덤 포레스트): 장점은 Ram을 많이 안먹으며, 훈련은 PC에서, 실제 코드는 조건문 가득이다.
      단점은 아두이노 모델에서는 훈련 불가능, 모델 크기가 너무 커지면 역시 RAM 요구가 많아진다
  - 데이터 로깅 및 시각화
    근거: 주행 중 센서 값의 변화를 그래프로 그리거나 파일로 저장하여 분석하는 작업은 파이썬의 시각화 도구(Matplotlib 등)가 압도적으로 유리합니다.
