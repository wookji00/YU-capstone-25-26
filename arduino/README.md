# 1. 아두이노(MCU) 단계: 실시간 제어 및 로우 레벨 필터링
  - 아두이노는 하드웨어와 직결되어 있어 반응 속도가 핵심인 알고리즘에 적합합니다.
    1. PID 제어 (모터 속도 및 방향 제어)
      근거: 모터의 회전수(RPM)를 맞추거나 균형을 잡는 일은 밀리초(ms) 단위의 빠른 피드백 루프가 필요합니다. PC(파이썬)를 거치면 통신 지연(Latency) 때문에 제어 타이밍을 놓쳐 차가 꿈뜨게 반응하거나 진동할 수 있습니다.
    2. 상보 필터 (Complementary Filter)
      근거: IMU의 노이즈를 제거하는 간단한 수식(더하기, 곱하기) 위주라 연산 부담이 적으면서도, 센서 원시 데이터를 즉각적으로 가공하여 제어 루프에 바로 던져줄 수 있습니다.
    3. 디바운싱 및 임계값 트리거 (정지 논리)
      근거: "조도가 일정 값 이하일 때 즉시 멈춤"과 같은 긴급 명령은 운영체제(OS)가 없는 아두이노에서 인터럽트(Interrupt) 방식으로 처리하는 것이 가장 빠르고 안전합니다.
